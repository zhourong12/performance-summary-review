# 自评内容审核标准

## 1. 充实度

### personalSummary

| 级别 | 条件 | 处理 |
|------|------|------|
| 空 | 空白或仅空格 | 不通过 |
| 极短 | < 20 字 | 不通过 |
| 过短 | 20–49 字 | 建议修改 |
| 合格 | ≥ 50 字且有实质内容 | 通过（需再做相关性检查） |

### 套话/模板化表述

以下模式若占 personalSummary 或 comment 主体（>60%），判为「空洞」：

- 「完成了本职工作」「表现良好」「继续努力」
- 「按时完成各项任务」「积极配合团队」
- 纯形容词堆砌，无具体事项、数据或结果

**处理**：建议修改，指出需补充具体工作内容和成果。

### selfReview[].comment

| 条件 | 处理 |
|------|------|
| 某指标 comment 为空 | 建议修改（列出缺失指标名） |
| 某指标 comment < 10 字 | 建议修改 |
| 全部 comment 为空或套话 | 不通过 |

## 2. 指标相关性

### personalSummary 与 indicators

对每个 `indicators[i]`：

1. 取指标名 `name` 的核心词（去掉「率」「数」「量」等后缀也可）
2. 检查 personalSummary 是否提及该指标相关工作、成果或困难
3. 覆盖率 = 有提及的指标数 / 指标总数

| 覆盖率 | 处理 |
|--------|------|
| 0% | 不通过（总结与指标完全脱节） |
| 1%–49% | 建议修改（列出未覆盖指标） |
| ≥ 50% | 通过（单项仍可能有弱相关） |

### selfReview comment 与 indicator

对每条 `selfReview[j]`，找到同名 `indicators` 项，对照 `description` / `criteria`：

| 判定 | 条件 |
|------|------|
| **相关** | comment 描述了该指标维度下的具体工作、结果或改进 |
| **弱相关** | 有文字但偏泛、未体现该指标考核点 |
| **缺失** | comment 为空 |
| **无关** | 内容与该指标 description/criteria 明显不符 |

- 任一指标「无关」→ 建议修改
- 超过半数「弱相关」或「缺失」→ 建议修改
- 全部「相关」→ 该项通过

## 3. 语义匹配方法

不必做字面关键词匹配，用语义理解：

- 「客户满意度提升」可匹配指标「客户投诉处理」
- 「完成了 3 个需求上线」可匹配指标「项目交付」
- 「参加了培训」若指标是「代码质量」，则判弱相关

## 4. 示例

### 通过

**personalSummary**（68 字）：
> 本月完成 A 项目核心模块开发，按时交付 2 个迭代；客户侧问题响应平均 4 小时内，投诉较上月降 30%；参与 1 次代码评审规范培训。

**selfReview**：
- 项目交付：comment 含具体项目名、交付节点 → 相关
- 客户响应：comment 含响应时效、投诉数据 → 相关

### 建议修改

**personalSummary**（32 字）：
> 本月工作认真，完成了领导交办的任务，团队协作良好。

**问题**：过短、套话、未覆盖具体指标。

### 不通过

**personalSummary**：`无` 或 `表现良好`

**selfReview**：全部 comment 为空或「已完成」

## 5. 批量审核优先级

批量时按严重程度排序输出：

1. 不通过（建议驳回）
2. 建议修改（personalSummary 过短）
3. 建议修改（指标未覆盖）
4. 通过

## 6. 驳回判定

| 审核结论 | 默认动作 |
|----------|----------|
| 不通过 | **建议驳回**；生成驳回原因草稿，待用户确认后调用 API |
| 建议修改 | 默认仅报告；用户说「驳回」「退回」时再走驳回流程 |
| 通过 | 不可驳回 |

### 常见驳回场景

- personalSummary 为空或「表现良好」等无实质内容
- 全部 selfReview comment 为空
- 总结与各指标工作完全无关
- 套话占比过高且无具体事项

### 不宜驳回

- 仅个别指标 comment 略短，但总结整体合格
- 用户仅要求「看看写得怎么样」未授权驳回
- 记录仍在 `self_review`（员工未提交）
- 当前账号不满足 integration 中的驳回角色（jixiao2：`super_admin`）
